> Created by Linda Cai @ Minus-AI | 2026

## 这是什么？

Evolution Protocol 是一套让你的 Claude Skill **自动进化**的共享协议。

普通 skill 是静态的——写好了就不会变。装了这个协议的 skill 会自动追踪进化方向、建立反馈闭环、用独立 agent 做质量对抗，像一个活的系统一样持续成长。

---

## 快速开始

### 安装（1 分钟）

把 `evolution-protocol.skill` 文件拖到 Claude Desktop 的 Settings → Skills 里，点击安装。安装后不需要手动操作，首次使用时会自动初始化。

### 两种用法

**用法 A：从零创建一个进化型 skill**

对 Claude 说：

> "帮我创建一个进化型 skill，用来 [你的目标]"

例如："帮我创建一个进化型 skill，用来追踪我的阅读成长"。Claude 会自动引导你定义目标函数、创建文件结构、生成所有必要文件。

**用法 B：把已有的 skill 改造为进化型**

对 Claude 说：

> "帮我把 [skill 名称] 改造成进化型 skill"

Claude 会读取你现有的 skill，按 Evolution Protocol 的标准架构重构：SKILL.md 瘦身为 bootloader，规则迁移到 RULES.md，新建 INDEX.md 和 SESSIONS.md，引导你定义目标函数。**改造过程会保留原 Skill 的全部内容**——原作者写的指令、规则、工作流程、领域知识一条不漏地迁移到新架构中，只增加进化机制，不删除或简化任何原有内容。

## 核心架构

### 解决什么问题

Claude Desktop 的 skill 安装后 SKILL.md 变成只读，但真正的"进化"需要可写。Evolution Protocol 用 **Bootloader 架构**解决这个矛盾：SKILL.md 只做引导（约 30 行），所有可进化的内容放在 workspace 里，Claude 可以自由读写。

### 文件结构

```
<skill-name>/
├── SKILL.md      ← 引导程序（只读，极少修改）
├── RULES.md      ← 规则大脑（可进化）
├── INDEX.md      ← 状态快照（每次对话同步）
├── SESSIONS.md   ← 对话日志（自动追加）
├── records/      ← 详细记录 + 裂变归档
├── scripts/      ← 可执行脚本
├── data/         ← 领域数据
└── outputs/      ← 产出物（可重建）
```

### 4 个必备文件，各有不可替代的作用

**SKILL.md（引导程序）**——装到 Claude Desktop 的只读文件。它只做三件事：告诉 Claude"我是谁"、"去哪里读规则"、以及万一规则文件找不到时的兜底规则。类比：电脑开机时的 BIOS，它不干活，只负责启动。

**RULES.md（规则大脑）**——所有详细规则：目标函数、工作流程、评分标准、领域专属逻辑。它是可进化的——Claude 可以自主做小调整，结构性变更会先问你。类比：一个公司的运营手册，随业务发展持续更新。

**INDEX.md（状态快照）**——不超过 150 行。包含：目标函数当前评分、最近 5 条关键记录、文件导航。每次新对话 Claude 只读这一个文件就能恢复全部状态。类比：一本书的目录——翻目录就知道讲了什么、翻到哪了，不用从头读。这是整个架构里最关键的效率设计。

**SESSIONS.md（对话日志）**——每次对话结束追加一条记录：主线、产出、决策、目标函数变动、上次建议的跟进结果、本次建议。超过 30 条自动归档。类比：带行动项回顾的会议纪要。

### 三个文件夹

**scripts/**（可执行脚本）、**data/**（领域数据，skill 运行依赖的放这里）、**outputs/**（产出物，可重建，分享时删除）。

### records/ 目录

放所有按需文件（DECISIONS.md、MILESTONES.md、CHALLENGE_LOG.md 等）和裂变归档。文件超过 30 条记录或 200 行自动裂变——旧的归档，活跃文件保持精简。

## 三大进化机制

### 1. 目标函数——定方向

每个进化型 skill 必须定义 3-7 个评分维度（推荐 1-5 分制），明确"进步"长什么样。创建 skill 时 Claude 会引导你定义 4 个要素：评分维度、打分尺度、评估频率、进化方向。

**没有目标函数的 skill 只是在"记录"，有了才是在"进化"。**

示例——读书笔记 skill 的目标函数：

| 维度 | 说明 | 初始分 |
|------|------|--------|
| 阅读深度 | 从摘要式→结构化→批判性阅读 | 2.0 |
| 跨领域连接 | 能否将不同书的观点串联起来 | 1.5 |
| 输出质量 | 笔记的清晰度和可复用性 | 2.0 |
| 持续性 | 阅读习惯的稳定程度 | 2.5 |

### 2. 反馈闭环——给动力

每次对话形成一个强化学习循环：

```
读取当前状态 → 检查上次建议是否被跟进 → 执行任务
→ 对照目标函数评估变化 → 生成下一步建议 → 写入记录
```

两个反馈信号源：**Human Feedback**（你可以修正 Claude 的评分，Claude 逐步校准向你对齐）和 **AI Self-Feedback**（Claude 自评上次建议的效果，连续 3 次未被采纳就调整策略）。

**停滞检测：** 某个维度连续 3 次对话没变化，Claude 主动提醒，帮你判断是目标太高、方法不对、还是优先级变了。

### 3. 左右互搏——保质量

**核心原则：builder 和 challenger 必须是两个独立的 agent。** 自己批改自己的作业会被自身盲点抵消。

三层设计，由轻到重：

**L1 嵌入式自检（每次自动）**——builder 在生成过程中自检基础规则。成本几乎为零。

**L2 独立 Challenger（每次自动）**——builder 完成后，启动独立 agent 逐条核对规则，输出具体违规项清单。成本约为 builder 的 20-30%。

**L3 策略级对抗（按需触发）**——不只查规则合规，还质疑规则本身是否合理。Claude 在特定情况下自动提醒（维度停滞、到达里程碑、规则大改），你确认后才执行。也可以随时说"challenge 一下"主动触发。

**高频失败清单**——challenger 发现的问题记录在 CHALLENGE_LOG.md。同类问题反复出现会自动进入重点狙击清单，不再出现的问题自动退出。这个清单本身也在进化——越用越精准。

## 完整举例：读书笔记 skill 的进化过程

小张装了 Evolution Protocol，说："帮我创建一个进化型 skill，追踪我的阅读成长。"

**创建阶段：** Claude 引导他定义目标函数（阅读深度 2.0、跨领域连接 1.5、输出质量 2.0、持续性 2.5），生成完整的文件结构，包括 SKILL.md、RULES.md、INDEX.md、SESSIONS.md。

**第 1 次对话：** 小张读完《思考快与慢》，Claude 帮整理笔记。评估：输出质量 2.0→2.5（结构化了），其他不变。L2 challenger 检查笔记，发现一个引用数据没标出处（🟡），builder 修正。建议："下次试着把'系统 1/系统 2'和你之前读的《原则》做交叉对比。"

**第 3 次对话：** Claude 发现"跨领域连接"连续 3 次停在 1.5，主动提醒。小张说换个方法。Claude 建议："每读完一本书，花 5 分钟写 3 个'这本书让我想到了哪本之前读过的书'。" "引用缺出处"进入高频失败清单。

**第 5 次对话：** 小张写了一篇串联 3 本书观点的笔记。评估：跨领域连接 1.5→2.5。Claude 问："你同意这个评分吗？" 小张说："我觉得 2.0 更合适，因为串联还比较浅。" Claude 校准，记录分歧。L2 challenger 检查——这次引用出处都有了，"引用缺出处"开始退出高频失败清单。

**这就是一个完整的进化循环——方向（目标函数）→ 动力（反馈闭环）→ 质量（左右互搏），三者协同让 skill 和用户一起成长。**

## 其他关键机制

**文件裂变：** 文件超 30 条或 200 行自动归档旧内容，保持活跃文件精简。SKILL.md、RULES.md、INDEX.md 永不裂变。

**紧急保存：** 上下文窗口快用完时，自动写入所有未保存记录。不允许丢失数据。

**跨 Skill 联动：** 多个进化型 skill 可以交叉引用，但各自维护各自的记录。

**共享协议升级：** 所有进化型 skill 启动时先读 evolution-protocol/RULES.md，改这一个文件所有 skill 同步升级，不需要逐个改。

**分享打包：** 保留规则和脚本，清空个人历史记录，删除产出物，扫描敏感信息后打包。

## 常见问题

**Q: 我需要懂编程吗？**
A: 不需要。安装 skill 后用自然语言和 Claude 对话就行。

**Q: 我可以修改规则吗？**
A: 可以。workspace 里的 RULES.md 完全可编辑。Claude 也会根据使用情况自主微调（小改动自动执行，大改动先问你）。

**Q: 能把已有的 skill 改造成进化型吗？**
A: 可以。对 Claude 说"帮我把 XX skill 改造成进化型 skill"，它会按标准架构重构。

**Q: 这个协议和我自己的 skill 是什么关系？**
A: Evolution Protocol 是"操作系统"，你的 skill 是跑在上面的"应用"。可以创建任意多个进化型 skill，都共享这套协议。

**Q: 如何更新协议？**
A: workspace 里的 RULES.md 会自动进化。如果有新版安装包，重新拖入 Claude Desktop 即可。
